# DESIGN.md

## Design Goal

Create a clean, trustworthy, human Croatian landing page for a small local IT help / computer repair service in Zagreb.

The page should make users feel:

- "This person is honest."
- "This is not a scam."
- "I understand what I can ask for."
- "The prices are clear."
- "I can send my problem without feeling stupid."
- "This is local and practical."

## Brand Direction

Brand idea:

**Direkt IT Pomoć Zagreb**

Possible emotional positioning:

> Poštena IT pomoć za obične ljude.

The design should feel like:

- a calm technician
- local Zagreb service
- practical help
- clear communication
- honest pricing
- beginner-friendly support

It should not feel like:

- a startup SaaS landing page
- a fake enterprise company
- a crypto/gaming/neon website
- an overdesigned agency template
- a generic AI-generated website

## Visual Keywords

Use these as design direction:

- clean
- local
- warm
- trustworthy
- grounded
- practical
- readable
- calm
- direct
- modern but not flashy
- premium but not corporate

## Mood

The mood should be:

- human
- clear
- warm
- reliable
- honest
- calm
- helpful

Avoid:

- aggressive
- flashy
- overly technical
- cold corporate
- fake premium
- childish
- overly futuristic

## Color System

Use a warm neutral base.

Recommended palette direction:

### Background

- warm off-white
- soft light gray
- subtle beige-gray
- not pure white everywhere

Example feel:

- `#F7F5F0`
- `#F5F6F4`
- `#FAFAF7`

### Text

Use deep charcoal, not pure black.

Example feel:

- `#171717`
- `#1F2933`
- `#222222`

### Primary Accent

Use calm blue, blue-gray, or blue-green.

Example feel:

- `#2563EB`
- `#1D4ED8`
- `#0F766E`
- `#256D85`

### Secondary Accent

Use soft amber only for small helpful notes or highlights.

Example feel:

- `#F59E0B`
- `#D97706`

### Borders

Use subtle neutral borders.

Example feel:

- `rgba(0,0,0,0.08)`
- `#E5E7EB`
- `#DDDAD3`

## Color Rules

Do:

- use calm contrast
- keep backgrounds soft
- use accent color for CTA and important highlights
- use borders/shadows subtly

Do not:

- use neon colors
- use neon gradients
- use heavy blue-purple AI gradients
- use too many colors
- make every card colorful
- use dark mode unless specifically requested
- overuse glassmorphism

## Typography

Typography should be readable and confident.

Headings:

- clear
- strong
- not too techy
- not too playful
- good line-height
- large enough on mobile

Body:

- comfortable
- human
- easy for older users
- not tiny
- not overly dense

Recommended direction:

- system font or clean sans-serif
- avoid overly stylized fonts
- use strong hierarchy

Suggested hierarchy:

```txt
Hero heading: very large, bold, tight but readable
Section heading: large, bold
Card title: medium, semibold
Body text: base, relaxed line-height
Small notes: readable, not too small
```

## Layout Principles

Use a single-page landing layout.

Preferred max width:

- content max width around `1120px` or `1200px`
- text sections narrower where needed
- generous horizontal padding on mobile

Spacing:

- use generous section padding
- keep consistent vertical rhythm
- avoid cramped cards
- avoid huge empty spaces

Cards:

- simple
- clear
- not overly rounded
- subtle border
- subtle shadow only if useful
- strong readability

Border radius:

- moderate
- avoid excessive "AI SaaS pill" look
- do not use huge rounded blobs everywhere

## Page Structure

Use this section order:

1. Hero
2. Common Problems
3. Services
4. Pricing Packages
5. How It Works
6. About Me
7. What I Do Not Do
8. FAQ
9. Contact CTA / Footer

## Section Design Direction

### Hero

The hero should immediately communicate:

- local IT help in Zagreb
- slow computers / Windows / backup
- honest pricing
- no complicated tech talk

Hero layout:

- strong headline on left or centered
- short subheadline
- primary CTA
- secondary CTA
- trust notes
- optional simple visual card on desktop

Avoid:

- fake laptop 3D illustration
- huge abstract gradient
- generic "modern solution" language
- too many badges

Good hero content:

```txt
Računalo sporo? Windows zeza? Trebate normalnu IT pomoć u Zagrebu?
```

CTA examples:

- Pošalji problem
- Nazovi / WhatsApp
- Pogledaj cijene

Trust note examples:

- Cijena dogovorena unaprijed
- Bez tehničkog preseravanja
- Ako ne znam, kažem odmah
- Zagreb i okolica

### Common Problems

Use cards or compact rows.

Problems should be written from the user's point of view:

- Računalo se sporo pali
- Programi se otvaraju predugo
- Windows izbacuje greške
- Ne znate što smijete obrisati
- Trebate backup važnih datoteka
- Trebate pomoć s mailom, browserom ili postavkama

Design:

- simple icons if used
- clear short text
- no long paragraphs

### Services

Use service cards with practical labels.

Suggested cards:

- Windows optimizacija
- Backup i prijenos podataka
- Instalacija programa
- Osnovna sigurnosna provjera
- Pomoć s internetom/mailom/browserom
- Savjetovanje prije kupnje

Each card should answer:

- what it is
- who it helps
- what result user can expect

Avoid:

- too much technical detail
- claiming advanced capabilities
- vague "IT solutions"

### Pricing Packages

Pricing is a trust section.

Design should be very clear.

Use 3 or 4 pricing cards.

Recommended cards:

1. Mini pomoć — od 20 €
2. Windows Refresh — oko 45 €
3. Backup + sređivanje — od 55 €
4. Kućna IT pomoć — po dogovoru

Pricing card rules:

- price visible
- short description
- bullet list of included work
- note if price depends on problem
- highlight the most common package

Most common package:

**Windows Refresh**

Design:

- one card can be slightly emphasized
- avoid aggressive "BEST VALUE!!!"
- use calm label like "Najčešći izbor"

### How It Works

Use a simple 5-step process:

1. Pošaljete problem
2. Dobijete kratku procjenu
3. Dogovorimo cijenu
4. Rješavam problem
5. Dobijete objašnjenje i savjet

Design:

- horizontal on desktop
- vertical on mobile
- numbered steps
- very short copy

### About Me

This section should build trust.

Tone:

- honest
- personal
- not arrogant
- not too long

Content direction:

- Mate / Vrachox
- Zagreb
- building career as IT technician / help desk / sysadmin
- practical IT orientation
- enjoys solving computer problems
- transparent about limits

Avoid:

- pretending to have huge company experience
- fake "team"
- fake certifications
- exaggerated experience

### What I Do Not Do

This section should be visible because it increases trust.

Include:

- no pirated software
- no advanced physical data recovery
- no motherboard repair
- no liquid damage repair
- no fake guarantee that everything can be fixed

Design:

- calm warning/info box
- not scary
- title like "Iskreno: što ne radim"

### FAQ

FAQ should answer practical concerns.

Design:

- accordion or simple list
- readable
- no tiny text
- no hidden critical information

FAQ examples:

- Radite li dolazak na lokaciju?
- Mogu li prvo poslati opis problema?
- Što ako se problem ne može riješiti?
- Radite li backup?
- Koliko traje sređivanje računala?
- Radite li piratski Windows ili Office?
- Je li cijena poznata unaprijed?
- Možete li pomoći starijim osobama?

### Contact CTA

This is the conversion section.

It should be simple and obvious.

Include:

- phone
- WhatsApp
- email
- Zagreb
- short instructions

Good CTA copy:

```txt
Pošaljite kratak opis problema i, ako možete, sliku greške. Odgovorit ću s realnom procjenom što se može napraviti i okvirnom cijenom.
```

Buttons:

- WhatsApp poruka
- Email
- Nazovi

If no functional form exists, do not fake it.

Use mailto/WhatsApp links instead.

## UI Component Rules

Buttons:

- clear label
- strong contrast
- not too rounded
- comfortable touch size
- primary and secondary style

Cards:

- simple border
- subtle background
- moderate radius
- no heavy shadows
- no excessive gradients

Icons:

- optional
- simple line icons
- do not overuse
- every icon should support meaning

Badges:

- use sparingly
- useful for trust notes or pricing labels
- avoid clutter

## Motion / Animation

Motion should be subtle.

Allowed:

- small fade-in
- small slide-up
- hover lift on cards
- subtle CTA hover

Avoid:

- large parallax
- spinning icons
- flashy transitions
- slow animations
- too many animated elements
- anything that distracts from contact CTA

If Framer Motion is used, keep it minimal.

## Mobile UX

Mobile is more important than desktop.

Mobile requirements:

- hero headline readable
- CTA visible without scrolling too much
- pricing cards easy to scan
- service cards not cramped
- contact buttons large enough
- no horizontal scroll
- no tiny FAQ text
- spacing comfortable

Recommended mobile CTA behavior:

- contact CTA near top
- optional sticky bottom contact bar only if it does not feel annoying
- WhatsApp/phone/email easy to tap

## Desktop UX

Desktop should feel polished but not empty.

Desktop requirements:

- good max-width
- section rhythm
- balanced hero
- cards aligned
- pricing readable
- no overly wide paragraphs
- footer/contact section clear

## SEO Design Considerations

Design should support SEO:

- one clear H1
- descriptive H2 sections
- visible text content
- FAQ content present in HTML
- not everything hidden behind interactions
- natural keywords in headings/copy
- contact/location visible
- fast-loading layout

## Accessibility Design

The design must be usable for non-technical and older users.

Requirements:

- high contrast
- readable font sizes
- clear buttons
- clear focus states
- no text over busy images
- no icon-only actions
- comfortable touch targets
- simple language

## Copywriting Style

Use Croatian.

Write like a real person.

Good copy traits:

- short sentences
- direct explanations
- practical examples
- no jargon unless explained
- honest limitations

Avoid:

- "digitalna transformacija"
- "najmodernija IT rješenja"
- "premium enterprise support"
- "revolucionarna optimizacija"
- "garantirano popravljamo sve"

## Anti-AI-Generic Rules

To avoid generic AI-looking design:

- do not use giant blue/purple gradients
- do not use vague hero illustration
- do not use buzzword headings
- do not use perfect fake testimonials
- do not make every card identical
- include honest "what I do not do" section
- include real local language
- include practical examples
- make pricing concrete
- use human microcopy

## Footer

Footer should include:

- service name
- Zagreb
- contact placeholders or real contact
- short disclaimer
- navigation links
- note about honest limitations

Suggested disclaimer:

```txt
Usluga je namijenjena osnovnim i srednje zahtjevnim IT problemima. Za fizička oštećenja, napredni data recovery i kompleksne hardverske kvarove preporučujem specijalizirani servis.
```

## Final Visual Acceptance Criteria

The design is good when:

- it feels like a real local service
- it does not look like a generic AI template
- the user understands the offer in 10 seconds
- pricing is easy to find
- contact is obvious
- mobile feels natural
- copy feels honest
- limitations are clear
- CTA is strong but not pushy
- the site feels clean, calm, and trustworthy
